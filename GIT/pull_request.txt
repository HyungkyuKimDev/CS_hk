プルリクエスト


プルリクエストとは
＞
　　プルリクエストとはGitHubの機能としてチーム開発に利用される機能だ。
　　プルリクエストを使わずにでデフォルトブランチにマージさればシステムに問題できる可能性があるので、
　　プルリクエストを通して変更の内容に問題があればその問題を解決し、作成されたコードよりいいコードを作るためにチームの中で共有して改善できる。
　　また、プルリクエストで送信されたソースコードに他の開発者がレビューを作成できる。
　　

プルリクエストの流れ
＞　
　　＃基本的な流れ　
　　ワークツリーで作成・変更で開発されたのをチームリーダーなどのレビュアーにプルリクエストを送信してリビューされた後デフォルトブランチにマージを行ってもらうようになる。
　　
　　＃１。リモートリポジトリをローカルリポジトリに複製する。
　　git colone <リモートリポジトリ＞

　　＃２。ブランチを作る。
　　git branch <作りたいブランチ名＞

　　＃３。ファイルの変更・コミット

　　＃４。リモートリポジトリにPushする。
　　git push origin ＜コミットされたローカルブランチ名＞

　　＃５。GitHubでプルリクエストを作成
　　プルリクエストの内容とコメントを作成
